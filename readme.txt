апи с доской объявлений. Как работает:
* Пользователя может добавлять только другой пользователь
* Добавлять и удалять записи на доску могут только пользователи
* По умолчанию есть один пользователь логин-пароль admin admin
* Что бы добавлять записи нужно сначала авторизоваться с логином и паролем
(получить токен и потом с ним уже делать остальные запросы)


Для запуска api нужно установить зависимости из requirements.txt и выполнить скрипт launcher.py.
После этого можно посмотреть запросы которые есть и данные которые они принимают тут http://localhost:9998/docs


Нужно набросать тесты которые будут проверять эту апи, как ты видишь. Для примера обложить тестами авторизацию:
1. Шаг добавить пользователя
2. Проверить что при указании правильного пароля выдается токен
3. Проверить что при указании неправильного пароля токен не выдается

Что еще можно проверить:
Тест того что с токеном можно делать действия которые нельзя делать без него
(а это практически все, кроме получения списка пользователей и просмотра "объявлений")

Придумай еще тесты глядя на документацию по запросам (http://localhost:9998/docs)

Плюсом будет если найдешь какие-то неправильные моменты по работе с апи
